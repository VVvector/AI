## 2. 配置说明

### 2.1 项目配置

#### package.json
```json
{
  "name": "todo-cli",
  "version": "1.0.0",
  "description": "A simple command-line todo list manager",
  "main": "src/index.js",
  "bin": {
    "todo": "src/index.js"
  },
  "scripts": {
    "test": "jest",
    "start": "node src/index.js"
  },
  "keywords": ["todo", "cli", "task-manager"],
  "author": "",
  "license": "MIT",
  "devDependencies": {
    "jest": "^29.7.0"
  }
}
```

### 2.2 安装配置

#### 本地开发安装
```bash
# 克隆仓库
git clone <repository-url>
cd todo-cli

# 安装依赖
npm install

# 链接到全局（可选）
npm link
```

#### 全局安装
```bash
# 从npm安装（如果已发布）
npm install -g todo-cli
```

### 2.3 使用配置

#### 数据文件位置
- 默认位置: `data/todos.json`
- 可以通过修改`fileHandler.js`中的`DATA_FILE_PATH`常量来自定义

#### 命令别名
- 可以在shell配置文件中添加别名以简化使用
  ```bash
  # Bash/Zsh
  alias t="todo"
  
  # PowerShell
  Set-Alias -Name t -Value todo
  ```

## 3. 常见问题和解决方案

### 3.1 数据文件损坏

**问题**: 数据文件损坏导致工具无法正常工作

**解决方案**:
1. 备份损坏的文件
2. 删除损坏的文件，工具会自动创建新的文件
3. 或者手动创建新的空数组文件

### 3.2 权限问题

**问题**: 无法读取或写入数据文件

**解决方案**:
1. 检查文件权限
2. 确保当前用户有读写权限
3. 可以尝试以管理员权限运行工具

### 3.3 命令不被识别

**问题**: 运行`todo`命令时提示"命令不被识别"

**解决方案**:
1. 确保已正确安装工具
2. 检查Node.js环境变量配置
3. 尝试使用`npx todo`命令
